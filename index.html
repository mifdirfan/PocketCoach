Project: PocketCoach

100% 비공개 AI 피트니스 코치

문제 정의 (The Problem)

기존의 건강 앱들은 사용자에게 효과성과 개인정보 보호 중 하나를 선택하도록 강요합니다. 사용자는 다음과 같은 주요 문제에 직면해 있습니다:

1) 특히 '찌개'와 같은 복잡한 한식의 경우, 수동으로 매크로를 추적하는 것은 매우 지루하여 사용자 이탈률이 높습니다.
2) 획일적인 운동 계획은 '회식'이나 개인 운동 기구 부족과 같은 실제 상황에 적응하지 못해 비효율적입니다.
3) 전문적인 인간 코칭은 매우 효과적이지만 대부분의 사람들에게 재정적으로 부담됩니다.
4) 클라우드 기반 앱은 사용자가 민감한 개인 건강 데이터를 업로드하도록 요구하여 심각한 개인정보 위험을 초래합니다.

PocketCoach의 목표는 100% 비공개로 로컬에서 실행되는 적응형 전문 AI 코치를 제공함으로써 이 네 가지 문제를 모두 해결하는 것입니다.

아키텍처/설계 (Architecture)

이 프로젝트는 100% 로컬, 개인정보 보호 우선의 클라이언트-서버 아키텍처를 기반으로 합니다.

• 백엔드 (The "Brain"): Python Flask 서버가 사용자의 로컬 PC에서 실행됩니다. Ollama LLM과 RAG 파이프라인을 관리하며, 모든 데이터는 로컬 JSON 파일에 저장됩니다.
• 프론트엔드 (The "Face"): Expo React Native(TypeScript) 앱이 UI 역할을 하며, 개인 WiFi 네트워크를 통해서만 로컬 Flask 서버와 통신합니다.
• RAG 시스템: master_food_db.csv(16만+), exercise_db.json 등 여러 데이터 소스로부터 구축됩니다.

핵심 기능 (Core Features)

RAG 기반 대화형 매크로 기록

자연어(예: "김치찌개 300g과 밥 한 공기")로 식단을 기록합니다. 백엔드는 Faiss-CPU를 사용해 16만+ 한식 데이터베이스에서 고속 검색을 수행합니다. 기록된 식사는 즉시 UI의 매크로 요약 테이블에 반영됩니다.

동적 및 적응형 계획 엔진

온보딩 시 AI가 개인화된 계획을 생성합니다. 사용자가 채팅으로 프로필(예: "체중 78kg으로 업데이트")을 변경하면, 백엔드가 user_profile.json을 수정하고 즉시 전체 계획을 재생성하여 최신 데이터에 완벽하게 적응합니다.

유연한 맞춤형 코칭

AI가 추천하는 모든 운동에 대해 YouTube 링크를 자동으로 검색하여 클릭 가능한 형태로 제공합니다. "오늘 30분짜리 운동으로 줘" 또는 "홈트레이닝만 할래"와 같은 실생활 요청에 즉시 적응하여 새로운 계획을 생성합니다.
